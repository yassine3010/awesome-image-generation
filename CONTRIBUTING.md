# Contributing

Thanks for your interest in contributing!

## Guidelines

- **One tool per PR** — keep changes focused and reviewable.
- **Format** — follow the existing entry format: `**Tool Name** – One-line description. [Website](link) | [Docs](link) | SDK: Python, Node`
- **API-first** — prioritize tools with developer APIs, SDKs, or hosted inference. Avoid raw research repos without integration paths.
- **Verify links** — ensure all URLs resolve before submitting.
- **Keep entries to 1-2 lines** — skimmable is the goal.
- **Correct section** — place your entry in the appropriate category.

## What We Accept

- Commercial image generation and editing APIs
- Open source models with hosted inference or pip install
- SDKs, CLIs, and developer tooling
- Image processing frameworks and UIs
- Infrastructure services relevant to image generation
- Templates, demos, and example projects
- Quality benchmarks and evaluation tools

## What We Don't Accept

- Research papers without usable code or API
- Paywalled tools without developer access
- Duplicate entries
- Self-promotional content without clear developer value

## Process

1. Fork the repo
2. Add your entry in the correct section
3. Submit a PR with a brief description of the tool
4. Ensure CI passes

Thank you!
